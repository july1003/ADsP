데이터베이스는 데이터를 독립적으로 관리할 수 있도록 설계되어 응용 프로그램으로부터의 독립성을 보장하는 것이 핵심 특징입니다.

1. 데이터의 최신성 보장
: 데이터베이스는 트랜잭션 처리와 동시성 제어 기능을 통해 실시간으로 데이터를 갱신하므로 항상 최신 상태의 데이터를 유지할 수 있습니다. 이는 데이터의 무결성과 정확성을 높이는 데 중요한 요소입니다.

2. 응용 프로그램 종속성
: 데이터베이스의 중요한 특징 중 하나는 데이터의 응용 프로그램으로부터의 독립성입니다. 즉, 응용 프로그램이 바뀌더라도 데이터 구조는 영향을 받지 않도록 설계되어야 하며, 이러한 독립성 덕분에 유지보수와 확장이 용이해집니다.

3. 데이터의 독립성 유지
: 데이터베이스는 논리적 독립성과 물리적 독립성을 제공하여, 사용자나 응용 프로그램이 데이터 구조를 변경하지 않고도 데이터를 활용할 수 있게 합니다.

4. 다중 사용자 지원
: 데이터베이스는 여러 사용자가 동시에 데이터에 접근하고 활용할 수 있도록 동시성 제어 및 잠금(Locking) 메커니즘 등을 통해 다중 사용자 환경을 지원합니다. 이는 데이터의 일관성을 유지하면서 효율적인 사용을 가능하게 합니다.

데이터 처리 및 분석은 현대 사회에서 필수적인 과정으로, 데이터를 단순 수치에서 의미 있는 정보로 변환하는 것이 중요합니다. 이를 통해 의사결정에 필요한 통찰을 얻을 수 있습니다.

1. 데이터 (Data)
: 데이터는 기본적으로 가공되지 않은 원시 자료입니다. 예를 들어, 온도 측정값이나 판매된 제품 수와 같은 단순한 숫자나 사실을 말합니다. 이러한 데이터는 자체로는 의미가 없으며, 해석과 분석을 통해 유용한 정보로 전환되어야 합니다. 데이터를 수집하고 저장하는 과정에서 주의할 점은, 데이터의 정확성과 일관성을 유지하는 것입니다.

2. 정보 (Information)
: 정보는 데이터를 분석하여 의미를 부여한 결과물입니다. 데이터 자체가 의미를 갖지 않지만, 이를 분석하고 패턴을 찾아내면 정보가 됩니다. 예를 들어, 하루 평균 기온 데이터를 분석하여 '다음 주에는 비가 올 확률이 높다'라는 정보를 도출할 수 있습니다. 정보는 의사결정의 기초가 되며, 정확한 정보를 얻기 위해서는 올바른 분석 기법과 도구가 필요합니다.

3. 지식(Knowledge)
: 지식은 정보를 활용하여 도출된 이해나 통찰입니다. 정보가 사람의 경험과 결합되면 특정 상황에서 유용한 결정을 내리는 데 도움을 주는 지식으로 발전합니다. 예를 들어, 반복적인 날씨 데이터를 통해 농부가 최적의 파종 시기를 결정하는 것이 지식의 활용입니다. 지식은 경험과 학습을 통해 축적되며, 이를 통해 문제 해결 능력을 키울 수 있습니다.

4. 지혜 (Wisdom)
: 지혜는 지식을 바탕으로 최적의 결정을 내리는 것을 의미합니다. 이는 단순히 사실을 아는 것을 넘어, 다양한 상황에 적용 가능한 통찰과 판단력을 포함합니다. 예를 들어, 복잡한 시장 상황을 분석하여 새로운 사업 기회를 창출하는 능력이 지혜에 해당합니다. 지혜를 키우기 위해서는 다양한 경험과 지속적인 학습이 필요합니다.

따라서, 정답은 '2. 정보 (Information)' 입니다.

SQL의 LIKE 연산자는 문자열 패턴을 비교할 때 사용되며, 다음과 같은 와일드카드를 사용합니다.
• _ (언더스코어): 임의의 한 글자를 의미합니다. 정확히 하나의 문자를 대체합니다.
• % (퍼센트): 0개 이상의 임의의 문자를 의미합니다. 아무 문자나 몇 글자든 상관없이 매칭됩니다.

주어진 SQL 문장을 해석해보면, e_customer_name의 첫 번째 문자는 어떤 글자든 가능하며, 두 번쨰 문자는 반드시 'A'이어야 하고, 세 번째 이후는 아무 글자나 0개 이상 가능하다는 것을 알 수 있습니다.

1. 영문명이 A로 시작하는 고객들의 이름
: 만약 영문명이 A로 시작하는 조건이라면 LIKE 'A%'를 사용해야 합니다. '_A%'는 첫 글자가 A여야 한다는 의미가 아니므로 이 설명은 부적절합니다.

2. 영문명에 A를 포함한 고객들의 비율
: 비율을 구하는 SQL 문이라면 COUNT나 SUM, AVG 같은 집계 함수와 비율 계산식이 있어야 합니다. 그러나 본 문장은 단순히 행을 선택(select)하는 구문이며, 비율을 계산하지 않습니다. 또한 출력 결과도 ‘고객명’과 ‘고객 영문명’이지, 비율은 출력되지 않습니다.

3. 위치 상관없이 영문명에 A를 포함하는 고객들의 이름
: 위치 상관없이 A가 포함되어야 한다면 LIKE '%A%'를 사용해야 합니다. 현재 사용된 패턴은 _A%로, 반드시 두 번째 글자가 A이어야 하므로 이 설명은 조건과 맞지 않습니다.

4. 영문명에 두 번째 문자가 A인 고객들의 이름
: _(언더스코어)는 첫 번째 글자를 의미하고, A는 두 번째 글자, %는 나머지를 의미하므로 이 조건은 두 번째 문자가 A인 e_customer_name을 필터링합니다. 이는 조건에 부합하는 설명입니다.

따라서, 문제의 정답은 '4. 영문명에 두 번째 문자가 A인 고객들의 이름' 입니다.

두 사건 A와 B가 독립이라면 A가 발생하더라도 B의 조건부 확률은 P(B)와 동일하다. 즉, P(B|A) = P(B)가 성립한다. 다른 선택지들은 확률론의 기본 개념으로, 사건의 수학적 확률, 상대도수의 수렴, 확률의 범위(0과 1 사이)는 모두 옳은 설명이다.

비기봇 해설
확률 및 확률분포의 핵심 개념은 사건의 확률 계산과 독립 사건의 조건부 확률입니다.

1. 사건 A가 일어나는 경우의 수를 모든 가능한 경우의 수로 나눈 값을 P(A)라 할 때 이를 A의 수학적 확률이라 한다.
: 수학적 확률은 확률의 기본 정의 중 하나로, P(A) = (사건 A가 일어나는 경우의 수)/(전체 경우의 수)로 계산됩니다. 예를 들어, 공정한 주사위를 던질 때 숫자 3이 나올 확률 P(A)는 1/6입니다.

2. 어떤 사건 A의 확률 P(A)가 주어졌을 때, n번의 실험에서 사건 A가 r번 발생했다고 할 때, 상대도수 r/n은 n이 커짐에 따라 P(A)에 근접하게 된다.
: 이 설명은 대수의 법칙(Law of Large Numbers)에 기반한 내용입니다. 대수의 법칙에 따르면, 실험 횟수 n이 증가할수록 상대도수 r/n은 사건 A의 실제 확률 P(A)에 수렴하게 됩니다. 예를 들어, 동전을 던졌을 때 앞면이 나올 확률이 P(A)=0.5라면, 실험 횟수를 증가시킬수록 앞면이 나온 횟수의 상대도수 r/n은 0.5에 근접하게 됩니다.

3. 두 사건 A, B가 독립일 때, 사건 B의 확률은 A가 일어났다는 가정 하에서의 B의 조건부 확률과는 다르다.
: 독립 사건이란, 두 사건 A와 B가 독립일 때, 사건 A의 발생 여부는 사건 B의 확률에 영향을 미치지 않는다는 것을 의미합니다. 즉, P(B|A) = P(B)가 성립합니다. 예를 들어, 주사위를 던질 때 첫 번째 던짐에서 3이 나왔다는 사건 A와 두 번째 던짐에서 5가 나왔다는 사건 B는 서로 독립입니다. 따라서 P(B∣A)는 단순히 P(B)=1/6과 동일합니다. 선택지 3번은 "조건부 확률이 다르다"고 설명하여 독립 사건의 정의에 위배되므로 부적절한 설명입니다.

4. 확률론에서 임의의 사건 A가 일어날 확률 P(A)는 항상 0 이상 1 이하의 값을 가진다.
: 확률 P(A)는 0과 1 사이의 값을 가지며, 이는 확률의 기본 성질입니다. P(A)=0일 경우에는 사건 A가 절대 발생하지 않는다는 의미이며, P(A)=1일 경우에는 사건 A가 항상 발생한다는 의미입니다. 예를 들어, 동전을 던질 때 "주사위에서 숫자 7이 나올 확률"은 0이고, "주사위를 던져 1에서 6 사이의 숫자가 나올 확률"은 1입니다.

사분위수 범위(Interquartile Range, IQR)는 전체 데이터에서 중간 50%에 해당하는 값들이 어느 정도 흩어져 있는지를 나타내는 지표입니다. 이는 3사분위수(Q3)에서 1사분위수(Q1)를 뺀 값으로 계산됩니다.

: 중앙값은 데이터의 중앙값 위치에 있는 값으로, 데이터의 중심 위치를 나타낼 뿐 데이터의 흩어진 정도(산포도)를 측정하는 값은 아닙니다.

2. 사분위수 범위(Interquantile Range)
: 사분위수 범위는 Q3 - Q1로 계산되며, 전체 데이터에서 가운데 50%가 얼마나 퍼져 있는지를 나타냅니다. 극단값에 영향을 받지 않고, 이상치 탐지나 분포 해석에 유용한 산포도 지표입니다.

3. 표준편차(Standard Deviation)
: 표준편차는 평균을 기준으로 데이터가 얼마나 퍼져 있는지를 나타내는 지표로, 전체 데이터의 흩어진 정도를 측정합니다. 하지만 문제에서는 “중앙 50%”에 국한된 범위를 묻고 있으므로 적절하지 않습니다.

4. 평균(Mean)
: 평균은 데이터의 중심 경향성을 나타내는 값이며, 산포도나 퍼짐 정도를 직접적으로 설명하지 않습니다.

따라서, 문제의 정답은 '2. 사분위수 범위(Interquantile Range)' 입니다.

신뢰구간을 계산할 때 빈칸에 들어갈 t-값은 신뢰구간의 폭을 결정하는 주요 요소입니다.

t-값은 표본 평균 주위의 범위를 정하며, 신뢰수준이 높아질수록 t-값이 커져 더 넓은 신뢰구간을 형성합니다.

• 표본 평균 (barX): 신뢰구간의 중심값으로, 데이터의 평균을 대표합니다.

• 표본 표준편차 (S): 데이터의 변동성을 나타내며, 신뢰구간 계산에서 표본의 불확실성을 반영합니다.

• 표본 크기 (n): 관측치 수를 의미하며, 표본이 클수록 신뢰구간의 폭이 줄어듭니다. 자유도는 n−1로 계산됩니다.

• 빈칸 (t-값): t-값은 자유도와 신뢰수준에 따른 누적 확률로 결정됩니다.
자유도 (df)는 표본 크기에서 1을 뺀 값으로, 이 문제에서는 관측치 71개에 따라 n−1=70으로 계산됩니다.

1. 히스토그램
: 히스토그램은 데이터의 분포를 시각화하는 데 유용한 방법입니다. 데이터가 대체로 대칭적이고 정규분포 형태(종 모양)를 가진다면 정규성을 만족한다고 볼 수 있습니다. 다만, 히스토그램은 시각적 확인이므로 정밀한 검증을 위해 다른 방법과 함께 사용해야 합니다.

2. Q-Q plot
: Q-Q plot은 데이터의 분위수를 이론적인 정규분포의 분위수와 비교하여 정규성을 확인하는 방법입니다. 만약 데이터가 정규성을 만족하면 Q-Q plot 상의 점들이 직선에 가깝게 위치합니다. 이는 데이터의 정규성을 시각적으로 평가할 수 있는 강력한 도구입니다.

3. Shapiro-Wilk test
: Shapiro-Wilk 검정은 데이터의 정규성을 검정하는 통계적 방법입니다. 귀무가설은 데이터가 정규분포를 따른다는 것이며, P-value가 유의수준보다 크면 정규성을 만족한다고 판단합니다.

4. Durbin-Watson test
: Durbin-Watson 검정은 데이터의 잔차가 독립적인지를 확인하는 데 사용됩니다. 회귀분석의 결과에서 잔차가 자기상관을 가지는지 여부를 확인하는 데 적합하며, 데이터의 정규성을 검정하는 데 사용되지 않습니다.

• q: 이동평균(MA, Moving Average) 항의 차수

• p=1: 자기회귀 항이 1차수
• d=2: 2번이 차분이 수행되었음을 의미
• q=3: 이동평균 항이 3차수

따라서, 정상성을 확보하기 위해 2번 차분이 수행되었으며, ARIMA 모형은 ARMA(1, 3) 모형으로 변환됩니다.

1. PCA 결과 이해
• Standard deviations (표준 편차): 주성분 축(PC1, PC2 등) 별로 데이터가 가지는 분산의 크기를 나타냅니다.
• Rotation matrix (회전 행렬): 각 변수(예: Murder, Assault 등)가 주성분 축에 기여하는 계수(즉, 가중치)를 나타냅니다.

2. 주성분 식 구하기
주성분은 원 데이터의 변수들을 선형 결합으로 표현하며, 공식은 아래와 같습니다.
PCk=c1k⋅X1+c2k⋅X2+⋯+cpk⋅Xp

• PCk: k번째 주성분 (PC1, PC2 등)
• c1k,c2k,⋯,cpk: 회전 행렬의 k번째 열에 해당하는 값(주성분의 가중치)
• X1,X2,⋯,Xp: 원 데이터의 변수들

PCA 결과에서 주성분의 계수는 회전 행렬의 값에서 가져오며, 주어진 PCA 결과의 회전 행렬은 아래와 같습니다.

비지도 학습(Unsupervised Learning)은 정답(label) 없이 데이터를 분석하여 숨겨진 구조나 패턴, 유사성을 찾아내는 분석 방법입니다. 대표적으로 군집분석, 연관규칙 분석, 차원축소 등이 이에 속합니다.
반면 지도 학습(Supervised Learning)은 정답(label)이 있는 데이터를 기반으로 예측 모형을 만드는 것으로, 분류와 회귀 분석이 이에 해당합니다.

가. 우편물에 인쇄된 우편번호 판별 분석을 통해 우편물을 자동으로 분류
: 우편번호는 정답(label)이 명확히 존재하므로, 이미 알고 있는 우편번호에 맞게 분류하는 작업입니다. 이는 분류(classification) 문제이며, 지도학습에 해당합니다.

나. 고객의 과거 거래 구매 패턴을 분석하여 고객이 구매하지 않은 상품을 추천
: 명시적인 ‘정답’이 없이, 패턴이나 유사성, 연관성을 기반으로 추천합니다. 주로 연관 규칙 분석(Apriori, FP-Growth), 협업 필터링 등의 비지도 방법이 사용됩니다.

다. 동일 차종의 수리 보고서 데이터를 분석하여 차량 수리에 소요되는 시간을 예측
: 목표 변수(정답)가 ‘소요 시간’으로 명확하게 존재하며, 수치 예측을 위한 회귀 분석(regression)이 필요한 문제입니다.

라. 상품을 구매할 때 그와 유사한 상품을 구매한 고객들의 구매 데이터를 분석하여 쿠폰을 발행
: 고객 간의 유사성이나 구매 패턴을 기반으로 잠재적인 군집이나 연관 규칙을 탐색합니다. 명확한 정답 없이 행동 기반의 유사성 분석이므로 비지도 학습에 해당합니다.

과대적합(Overfitting)은 모델이 훈련 데이터에 너무 잘 맞도록 학습된 나머지, 새로운 데이터(테스트 데이터)에는 일반화 성능이 떨어지는 현상을 의미합니다.

1. 일반화
: 일반화는 모형이 훈련 데이터뿐만 아니라 새로운 데이터(예: 테스트 데이터, 실전 데이터)에도 잘 작동하는 능력을 말합니다. 그러나 문제에서 주어진 상황에서는 훈련 데이터에서는 성능이 높지만 테스트 데이터에서는 성능이 급격히 저하되므로, 이는 일반화에 실패한 경우입니다.

2. 과대적합
: 과대적합은 모델이 훈련 데이터에 지나치게 민감하게 반응하여, 노이즈까지 학습해버리는 현상입니다. 그 결과, 모델은 훈련 데이터에서는 높은 성능(예: 높은 적중률)을 보이지만, 테스트 데이터나 실 운영 환경에서는 성능이 급격히 떨어지는 문제가 발생합니다.

3. 미적합
: 미적합은 모델이 너무 단순하거나 훈련이 부족하여 데이터의 기본적인 패턴조차 학습하지 못하는 상태를 의미합니다. 이 경우에는 훈련 데이터와 테스트 데이터 모두에서 성능이 낮게 나옵니다. 문제에서는 훈련 데이터에서는 적중률이 높다고 했기 때문에, 이는 미적합에 해당되지 않습니다.

4. 과소평가
: 과소평가는 일반적으로 어떤 변수의 효과, 위험도, 비용 등을 실제보다 작게 평가하는 판단 오류를 의미합니다. 이는 통계적 검정이나 의사결정 평가의 개념이지, 모형의 학습 성능이나 적중률 저하 현상과는 직접적인 관련이 없습니다.

따라서, 문제의 정답은 '2. 과대적합' 입니다.

비기봇 해설
정답은 1번입니다.

민감도(Sensitivity)는 실제 양성 중에서 모델이 양성으로 예측한 비율을 나타내는 지표입니다. 이는 모델이 실제 양성을 얼마나 잘 예측하는지를 평가하는 데 사용됩니다.

1. TP / (TP + FN)
: 민감도(Sensitivity, Recall)는 실제 양성(Positive) 사례 중에서 양성으로 정확하게 예측된 비율을 의미합니다. 여기서 TP는 실제 양성을 양성으로 예측한 건수, FN은 실제 양성을 음성으로 잘못 예측한 건수입니다.

2. TP / (TP + FP)
: 이 공식은 정밀도(Precision)를 계산하는 식입니다. 정밀도는 양성으로 예측한 사례 중에서 실제 양성인 비율을 의미합니다.

3. TN / (TN + FP)
: 이 공식은 특이도(Specificity)를 계산하는 식입니다. 특이도는 실제 음성 사례 중에서 음성으로 정확히 예측한 비율을 의미합니다.

4. FP / (FP + FN)
: 이 식은 주요 성능 지표의 공식에 해당하지 않으며, 잘못된 식입니다. FP와 FN은 서로 다른 오류 유형(거짓 양성, 거짓 음성)으로 조합되어 있어, 성능 지표로서 의미를 가지지 않습니다.

따라서, 문제의 정답은 '1. TP / (TP + FN)' 입니다.

연관 분석은 데이터 내의 항목 간 관계를 탐색하는 데 유용하며, 주로 대규모 데이터에서 규칙을 발견하는 데 사용됩니다.

1. 결과가 조건 반응(if-then) 형태로 표현되어 해석이 용이하다.
: 연관 분석의 핵심 장점은 결과를 "A → B" 형태로 제공하는 조건 반응(if-then) 규칙으로 표현된다는 점입니다. 예를 들어, "빵을 구매한 고객이 우유를 구매할 확률은 60%"와 같은 직관적인 규칙을 제공합니다. 이러한 표현 방식은 비전문가도 쉽게 이해할 수 있으며, 규칙의 실질적 활용 가능성을 높입니다.

2. 분석에 목적변수가 필요 없으므로 다양한 데이터에 적용할 수 있다.
: 연관 분석은 비지도 학습 방법으로, 분석 전에 종속변수(목적변수)를 정의할 필요가 없습니다. 따라서 데이터가 갖고 있는 관계를 자유롭게 탐색하고 숨겨진 패턴을 발견하는 데 적합합니다. 예를 들어, 고객의 구매 이력을 활용하여 특정 품목 간의 관계를 탐색하거나, 의료 데이터에서 질병 간 연관성을 분석할 수 있습니다.

3. 품목 세분화에 관계없이 의미 있는 규칙 발견이 가능하다.
: 연관 분석에서 품목 세분화의 수준은 분석 결과의 품질에 직접적인 영향을 미칩니다. 품목을 과도하게 세분화하면 데이터의 희소성이 증가하여 빈발 항목 집합을 찾기 어려워지고, 과도하게 일반화하면 규칙의 구체성이 낮아질 수 있습니다. 예를 들어, "전자제품"이라는 카테고리로 분석할 때보다 "스마트폰"과 같은 세부 항목으로 분석할 때 더 유용한 규칙이 발견될 가능성이 큽니다.

4. 계산 과정이 비교적 간단하여 대규모 데이터에도 적용 가능하다.
: 연관 분석은 Apriori, Eclat 등의 알고리즘을 사용하여 빈발 항목 집합을 탐색하고 연관 규칙을 생성합니다. 이 과정은 지지도(Support)와 신뢰도(Confidence) 등의 기준값을 기반으로 작동하며, 계산 과정이 효율적으로 설계되어 대규모 데이터에도 적용 가능합니다. 예를 들어, 대형마트의 수백만 건의 거래 데이터를 분석하여 고객 구매 패턴을 발견할 수 있습니다. 이는 연관 분석이 대규모 데이터 환경에서도 활용 가능한 중요한 이유입니다.

3과목 연관분석 오답
50. 아래의 거래 내역에서 연관규칙 사과 → 딸기에 대한 향상도로 옳은 것은?
  1. 0.5/(0.7 × 0.45)
  2. 0.3/(0.2 × 0.1)
  3. 0.3/(0.7 × 0.45)
  4. 0.1/(0.2 × 0.1)
정답보기
정답
3 0.3/(0.7 × 0.45)
해설
향상도(Lift)는 다음과 같이 계산된다. Lift(A→B)= Support(A∩B) / Support(A)×Support(B) 해당 공식에서 A에는 사과, B에는 딸기를 대입하여 풀이할 수 있다. A∩B = 0.3, A = 0.7, B = 0.45 이므로 향상도를 옳게 계산한 식은 3번이다.

비기봇 해설
연관규칙 분석은 거래 데이터에서 특정 항목 간의 관계를 분석하는 기법으로, 지지도(Support), 신뢰도(Confidence), 향상도(Lift)와 같은 주요 지표를 통해 규칙의 유효성을 평가합니다.
계산 과정은 아래와 같습니다.

1. 사과와 딸기의 지지도
사과와 딸기가 동시에 포함된 거래 수는 20 + 40 = 60이고, 전체 거래는 200이므로 지지도를 계산하면 아래와 같습니다.
Support(사과, 딸기) = 60/200 = 0.3

2. 사과의 확률
사과가 포함된 모든 거래 수는 40 + 20 + 40 + 40 = 140이고, 전체 거래는 200이므로 사과의 확률은 아래와 같습니다.
P(사과) = 140/200=0.7

3. 딸기의 확률
딸기가 포함된 모든 거래 수는 20 + 20 + 10 + 40 = 90이고, 전체 거래는 200이므로 딸기의 확률은 아래와 같습니다.
P(딸기) = 90/200=0.45

4. 향상도(Lift)
향상도는 사과가 구매되지 않았을 때 딸기의 구매 확률에 비해 사과가 구매 돼고 딸기의 구매확률의 증가를 표현한 것입니다. 식은 아래와 같습니다.
Lift(사과 -> 딸기) = support(사과 -> 딸기)/p(사과)*p(딸기) = 0.3/(0.7*0.45) =0.95
