2. 여행회사
: 여행회사는 고객 예약 정보, 여행 선호도, 계절별 여행 패턴 등 다양한 데이터를 확보할 수 있습니다. 고객 행동 데이터 기반의 맞춤형 서비스와 마케팅을 진행할 수 있어 빅데이터를 통한 경쟁력 강화가 가능합니다.

3. B2B기업
: B2B기업은 기업 간 거래를 중심으로 하며, 개인 소비자 데이터 확보가 어렵습니다. 거래 대상이 소수의 기업에 국한되기 때문에 다양한 비정형 데이터를 대량으로 수집하거나 분석하는 데 한계가 있습니다.

4. 이동통신사

2. 유튜브 영상의 썸네일 사진
: 썸네일 사진은 이미지 데이터이며, 비정형 데이터에 해당합니다. 이미지 파일은 픽셀과 색상 정보로 저장되며, 표 형태의 데이터로 쉽게 변환할 수 없습니다. 분석을 위해서는 컴퓨터 비전 기술이나 OCR(광학 문자 인식) 등의 기법을 사용해야 합니다.

3. 유튜브 영상의 좋아요 수
: 유튜브 영상의 좋아요 수는 숫자로 표현되며, 정형 데이터입니다. 일반적으로 좋아요 수는 정수 값으로 저장되며, 데이터베이스의 테이블에서 열(Column) 형태로 관리할 수 있습니다. 또한, 평균, 합계 등의 연산이 용이합니다. 따라서 비정형 데이터가 아니며 정답입니다.

개인정보 비식별화 기술은 개인정보를 식별할 수 없도록 가공하는 방법을 말합니다. 문제에서 설명하는 '주요 식별요소를 다른 값으로 대체하여 개인 식별을 어렵게 만드는 기술'은 본래 정보를 유지하되, 다른 값으로 치환하는 방식을 의미하며, 이는 가명처리에 해당합니다.

1. 가명처리(Pseudonymization)
: 가명처리는 개인정보 주체의 이름이나 식별정보를 다른 값으로 대체하여 직접 식별이 어렵도록 만드는 기술입니다. 일정한 규칙에 따라 치환되며, 별도로 관리되는 정보를 통해서만 원래 정보를 복원할 수 있습니다. 문제에서 말하는 '다른 값으로 대체하여 식별을 어렵게 만드는' 설명에 정확히 부합합니다.

2. 데이터삭제(Data Reduction)
: 데이터삭제는 개인을 식별할 수 있는 주요 정보를 아예 제거하여 비식별성을 확보하는 방법입니다. 치환이 아니라 삭제하는 것이므로, 문제에서 설명하는 치환 방식과는 다릅니다.

3. 범주화(Data Suppression)
: 범주화는 데이터를 구체적인 수치나 값 대신 범위나 그룹으로 변환하는 방법입니다. 예를 들어, 나이 35세를 '30~40세' 범주로 묶는 방식입니다. 데이터의 구체적인 값을 바꾸는 것이 아니라 범주로 숨기는 방식이기 때문에 문제에서 설명하는 대체 방법과 다릅니다.

4. 데이터마스킹(Data Masking)
: 데이터마스킹은 데이터를 읽을 수 없도록 일부를 가리거나 변형하는 기술입니다. 예를 들어 이름 중 일부를 '*'로 처리하는 방식입니다. 이는 데이터의 일부를 가리는 방법이므로 치환과는 다릅니다.

라. 커뮤니케이션 능력

분석은 대상(What)과 방법(How)에 따라 최적화(Optimization), 솔루션(Solution), 통찰(Insight), 발견(Discovery) 네 가지 유형으로 나뉩니다. 각각의 경우에 따라 적절한 분석 접근 방식이 필요합니다.

: 분석 대상과 방법이 모두 모호한 경우에는 인사이트(insight)를 찾는 것이 아니라 발견(discovery)이 이루어져야 합니다. 인사이트는 분석 대상이 불명확하지만 기존 분석 방법을 활용할 수 있을 때 가능한 방식이며, 분석 대상과 방법이 모두 모호하다면 새로운 패턴을 탐색하는 발견 과정이 필요합니다.

3. 분석 방법만 알면 인사이트를 찾을 수 있다.
: 분석 대상이 명확하지 않더라도 기존의 분석 방법을 적용하여 새로운 패턴을 찾는 과정이 인사이트(insight)입니다. 예를 들어, 특정 제품의 판매 데이터가 부족하지만, 머신러닝 모델을 적용하여 시장의 패턴을 분석할 수 있습니다.

4. 대상과 분석 방법을 알면 최적화가 가능하다.
: 분석 대상과 방법이 모두 명확하면 최적의 분석 기법을 적용하여 효율성을 극대화할 수 있습니다. 예를 들어, 공장 자동화 데이터가 명확하고 분석 기법도 정해져 있다면, 이를 활용하여 공정 최적화를 수행할 수 있습니다.

정답은 3번입니다.

KDD 분석 절차는 데이터 선택, 전처리, 변환, 데이터 마이닝, 평가의 다섯 단계로 이루어집니다. 문제에서 제시된 '분석 목적에 맞게 변수를 생성, 선택하고 차원을 축소하는 과정'은 데이터 변환 단계에 해당합니다.

1. 데이터 선택(Selection)
: 데이터 선택은 분석할 데이터를 외부 데이터베이스나 원시 데이터 중에서 선별하는 과정입니다. 필요한 데이터셋을 선택하는 단계로, 변수를 새로 생성하거나 차원을 축소하는 작업은 수행하지 않습니다.

2. 데이터 전처리(Preprocessing)
: 데이터 전처리는 수집된 데이터에서 잡음, 이상치, 결측치 등을 제거하거나 수정하여 데이터를 정제하는 과정입니다. 데이터 품질을 높이는 단계이며, 변수를 새로 만들거나 차원을 축소하는 것은 이 단계의 주요 목표가 아닙니다.

3. 데이터 변환(Transformation)
: 데이터 변환은 정제된 데이터를 분석 목적에 맞게 변형하는 과정입니다. 구체적으로는 필요한 변수를 새로 생성하거나 선택하고, 데이터 차원을 축소하여 효율적으로 마이닝할 수 있도록 데이터를 구조화합니다. 문제 지문에 나온 '변수를 생성, 선택하고 데이터 차원을 축소하는' 설명에 정확히 부합하므로 정답입니다.

4. 모델링(Data Mining)
: 데이터 마이닝은 변환된 데이터를 이용하여 특정 분석 목적에 맞는 알고리즘을 적용하고, 모델을 구축하는 과정입니다. 데이터 변환이 완료된 이후에 모델을 만드는 단계이므로, 변수 생성이나 차원 축소는 이 단계의 주요 작업이 아닙니다. 따라서 부적절합니다.

따라서, 문제의 정답은 '3. 데이터 변환(Transformation)' 입니다.

: 하향식 접근에서는 기업의 내·외부 환경을 포괄적으로 분석하여 해결이 필요한 비즈니스 문제를 도출합니다. 외부 시장 동향, 경쟁사 사례, 고객 행동 등 외부 요인뿐만 아니라 내부의 비즈니스 프로세스, 전략 등도 함께 고려하여 문제를 탐색합니다.

2. 기업내부의 과거 데이터를 무조건 결합 및 활용한다.
: 기업 내부의 과거 데이터를 무조건 결합 및 활용하는 것은 하향식 접근법과 맞지 않으며, 비즈니스 환경과 분석 목적에 부합하는 기업의 과거 데이터를 활용해야 합니다.

3. 식별된 비즈니스 문제를 데이터의 문제로 변환하여 정의한다.
: 비즈니스 관점에서 식별된 문제를 구체적인 데이터 문제로 변환하는 작업을 의미합니다. 이를 통해 데이터 분석이 비즈니스 목표와 직접적으로 연계될 수 있도록 합니다.

4. 도출된 분석 문제나 가설에 대한 대안을 과제화하기 위해 타당성을 평가한다.

: 대체재는 경쟁자 확대 관점에 해당하며, 시장 니즈 탐색 관점에는 포함되지 않습니다. 경쟁자 확대 관점에서의 대체재는 융합적인 경쟁 환경에서 현재 생산 중인 제품·서비스의 온라인 제공에 대한 탐색 및 잠재적 위협 파악하는 영역입니다.

상향식 접근방식은 데이터 분석을 통해 문제를 찾아가는 과정으로, 주어진 사물을 '있는 그대로(What)' 인식하고, 패턴이나 관계를 발견하는 데 초점을 둡니다. 분석적으로 'Why'를 탐색하는 것은 명확한 문제 설정에서 출발하는 하향식 접근방식의 특징입니다.

2. 분석적으로 사물을 인식하려는 'Why' 관점에서 접근한다.
: 'Why' 관점은 원인과 결과를 파악하려는 분석적 접근이며, 이는 하향식(top-down) 접근방식의 특징입니다. 상향식 접근방식은 'What' 관점, 즉 사물을 있는 그대로 관찰하는 데 초점을 맞추기 때문에, 이 설명은 옳지 않습니다.

: 전통적으로 데이터 분석은 인과관계 중심이었으나, 상향식 접근방식에서는 인과관계보다 상관관계를 통해 의미를 찾으려는 경향으로 변화하였습니다.

4. 사물을 있는 그대로 인식하는 'What' 관점에서 접근한다.

따라서, 문제의 정답은 '2. 분석적으로 사물을 인식하려는 'Why' 관점에서 접근한다.' 입니다.

분석 성숙도 모델은 분석이 조직 내에서 얼마나 성숙하게 활용되고 있는지를 나타내는 단계적 모델입니다. 도입단계 → 활용단계 → 확산단계 → 최적화단계로 발전하며, 각 단계별로 분석을 수행하는 방식과 활용 수준이 달라집니다.

1. 분석 COE 조직 운영
: 분석 COE(Center of Excellence) 조직 운영은 분석이 전사적으로 확산되는 과정에서 등장하는 개념으로, 확산단계에 속합니다. 이 단계에서는 전사 모든 부서가 분석을 수행하고, 데이터 사이언티스트를 확보하여 체계적인 분석 환경을 구축합니다. 선택지 중 유일하게 "확산단계"에 속하는 내용이므로, 다른 단계와 차이가 있습니다.

2. 전문 담당부서에서 수행
: 전문 담당부서에서 수행은 활용단계에 속합니다. 도입단계에서는 일부 부서에서 분석을 수행하지만, 활용단계에서는 분석이 특정 전문 부서에서 수행됩니다. 이 단계에서는 데이터 분석의 전문성이 증가하며, 특정 부서에서 분석을 주도하는 역할을 하게 됩니다.

3. 분석기법 도입

비기봇 해설
이 문제는 정밀도(Precision)를 계산하는 문제입니다. 정밀도는 "신용 불량으로 진단된 사람 중 실제로 신용 불량인 사람의 비율"을 의미합니다.

1. 주어진 데이터
• P(실제 신용 불량) = P(A) = 15% = 0.15
• P(신용 불량으로 진단) = P(B) = 25% = 0.25
• P(진단 = 불량 | 실제 불량) = 80% = 0.8 (민감도, Sensitivity)

2. 정밀도 계산
정밀도(Precision)는 아래와 같이 계산됩니다.
P(A|B)=P(B|A)⋅P(A)P(B)

주어진 데이터들을 해당 공식에 대입하여 계산하면 아래와 같습니다.
P(A|B)=0.8⋅0.150.25=0.120.25=0.48=48%

즉, 신용 불량으로 진단된 사람이 실제로 신용 불량일 확률은 2. 48%입니다.

3. R²의 값이 클수록 회귀선으로 실제 관찰치를 예측하는데 정확성이 높아진다.

해설
Fb 지표는 precision과 recall의 가중치를 조정하여 모델 성능을 평가하는 지표이다. Fb 값이 커질수록 recall의 중요도가 증가한다.

비기봇 해설
Fb 지표의 공식은 다음과 같습니다.
Fβ=(1+β2)×precision×recall(β2×precision)+recall

1. Fb가 0일 경우, precision과 recall이 동일한 가중치를 갖는다.
:F0=(1+02)×precision×recall(02×precision)+recall
=precision×recallrecall=precision
Fb가 0일 때는 F-measure가 precision과 같아지며, recall은 전혀 고려하지 않습니다. 즉, precision에만 가중치를 둡니다. 따라서 precision과 recall이 동일한 가중치를 갖는다는 설명은 틀렸습니다.

2. Fb가 2일 경우, recall에 2배 가중치를 두어 평균한다.
: Fb가 2일 경우, β = 2 입니다. 공식에 대입하여 풀이해보면 β² = 4이므로, precision보다 recall이 4배 더 많은 가중치를 받습니다.

3. Fb가 0.5일 경우, precision에 2배 가중치를 두어 평균한다.
: Fb가 0.5일 경우, β = 0.5 입니다. β < 1 이면 precision에 recall보다 더 큰 가중치를 두는 것은 맞지만, 정확히 2배의 가중치를 둔다고 단정하기는 어렵습니다. 마찬가지로 Fb 값은 가중치의 비율을 조절하는 역할을 하며, 단순하게 2배 가중치를 둔다고 표현하기에는 무리가 있습니다.

4. Fb가 커질수록 recall의 비중이 증가한다.
:Fb 값이 커진다는 것은 β 값이 커진다는 의미입니다. β 값이 커질수록 공식에서 recall이 결과값에 미치는 영향이 커집니다. 즉, recall에 더 큰 가중치를 두게 됩니다. 따라서 Fb가 커질수록 recall의 비중이 증가한다는 설명은 옳습니다.

연관분석은 거래 내 여러 항목 간의 연관규칙을 찾는 분석 기법으로, 대표적인 측도로 지지도(support), 신뢰도(confidence), 향상도(lift) 등이 있습니다. 그중 신뢰도는 조건부 확률로 해석되며, 어떤 항목 A가 발생했을 때 항목 B가 함께 발생할 확률을 의미합니다.

Support(A→B)=count(A∩B)N
이는 A와 B가 동시에 나타나는 빈도를 전체 거래 수로 나눈 값입니다.

Confidence(A→B)=count(A∩B)count(A)
이는 A가 발생했을 때 B가 함께 발생할 조건부 확률이며, 문제에서 제시한 "A를 포함한 거래 중 B가 포함될 확률"과 정확히 일치합니다.

3. 특이도

4. 민감도
: 민감도는 이진분류에서 실제 양성(Positive) 중에서 모델이 양성으로 정확히 예측한 비율을 말합니다. 마찬가지로 연관분석과는 관련이 없습니다.
