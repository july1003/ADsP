데이터는 정형(Structured), 반정형(Semi-Structured), 비정형(Unstructured) 데이터로 구분되며, 활용 방식에 따라 저장 방식과 처리 방법이 다릅니다. 또한, 데이터의 가치는 단순히 양이 많다고 해서 결정되는 것이 아니라, 의미 있는 정보로 변환될 수 있는지가 핵심 요소입니다.

1. 1킬로바이트는 1024바이트로 구성되며, 1바이트는 256종류의 값을 표현할 수 있다.
: 1바이트(Byte)는 8비트(bit)로 구성되며, 8비트는 2⁸ = 256가지의 값을 표현할 수 있습니다. 또한, 1킬로바이트(KB)는 정확히 1024바이트로 구성됩니다.

2. 텍스트 데이터를 데이터베이스에 저장하는 것은 수치 데이터를 저장하는 것보다 더 많은 자원과 시간이 소요된다.
: 텍스트 데이터는 수치 데이터보다 저장 크기가 크고, 검색 및 인덱싱 과정이 더 복잡하여 데이터베이스에서 저장 및 처리하는 데 더 많은 자원과 시간이 소요됩니다.

3. 데이터가 많을수록 더 많은 가치가 창출된다.
: 데이터가 많다고 해서 반드시 더 많은 가치를 제공하는 것은 아닙니다. 중복 데이터, 불필요한 데이터, 노이즈 데이터가 많아지면 분석 과정이 복잡해지고 오히려 정확성이 떨어질 수 있습니다. 따라서 데이터의 품질과 적절한 활용이 중요합니다.

4. 이미지 파일은 구조화된 형식으로 저장되지 않기 때문에 비정형 데이터에 해당한다.
: 이미지, 영상, 음성 데이터는 일정한 테이블 형식으로 저장되지 않고, 별도의 메타데이터와 함께 저장되는 비정형 데이터(Unstructured Data)에 속합니다.

: 데이터가 커질수록 저장·처리·분석 과정에서 시간과 자원이 더 소모되므로 분산 처리 시스템, 병렬처리 알고리즘, 클라우드 인프라 등이 필수적입니다.

2. 데이터의 품질은 데이터 정제와 전처리 과정을 통해 분석 결과의 신뢰도를 높이는 데 중요한 역할을 함

3. 비정형 데이터는 데이터 내부에 메타데이터를 갖고 있으며 일반적으로 파일 형태로 저장됨
: 비정형 데이터는 구조화되지 않은 형태의 데이터로, 반드시 메타데이터를 포함하거나 파일 형태로 저장되는 것은 아닙니다. 예를 들어 웹 크롤링으로 수집한 데이터나 센서 로그는 텍스트 형태일 수 있으며, 메타데이터가 따로 존재하지 않거나 외부에 저장되기도 합니다.

4. 데이터의 상관관계를 분석하면 숨겨진 패턴이나 유용한 인사이트를 발견할 수 있음
: 상관관계 분석은 두 변수 간의 연관 정도를 파악하는 기초 분석 기법으로, 통계적 상관계수(예: 피어슨 상관계수, 스피어만 순위상관계수)를 이용해 수치적으로 표현합니다. 또한, 변수들 간 관계를 시각화하거나 회귀분석, 클러스터링 등 고급 분석의 사전 단계로서도 활용됩니다.

데이터는 정형 데이터와 비정형 데이터로 구분할 수 있습니다.

비기봇 해설
정답은 1번 입니다.

데이터 유형은 데이터의 구조화 정도에 따라 정형, 반정형, 비정형 데이터로 나뉩니다. 정형 데이터는 고정된 필드와 구조를 가진 데이터로, 주로 데이터베이스나 스프레드시트에 저장됩니다. 반정형 데이터는 일정한 형식이 있지만 유연한 구조를 가지며, XML, JSON, SNS 데이터 등이 이에 해당합니다. 비정형 데이터는 정해진 형식이 없으며, 텍스트, 이미지, 위치 데이터 등이 포함됩니다.

1. Demand Forecasts – Competitor pricing – Email records
: Demand Forecasts는 날짜, 제품 ID, 예측 수량 등 고정된 스키마와 수치로 구성되어 정형 데이터베이스에 저장되는 정형 데이터이며, Competitor pricing은 제품명·가격·통화 등의 필드를 포함하되 출처에 따라 구조가 유동적인 JSON 또는 XML 형태로 제공되는 반정형 데이터, Email records는 본문 내용과 첨부 파일 등 자유로운 텍스트 및 미디어 형태로 정해진 구조 없이 저장되는 비정형 데이터입니다.

2. Facebook status – Weather data – Web logs
: Facebook status(페이스북 상태 메시지)는 자유 형식의 텍스트로 구성된 비정형 데이터이며, Weather data(기상 데이터)는 기온·강수량 등 고정된 수치 필드를 가진 정형 데이터, Web logs(웹 로그)는 JSON이나 특정 로그 파일 형식으로 저장되는 반정형 데이터입니다. 이는 반정형–정형–비정형 순서로, 정형–반정형–비정형 순서를 따르지 않습니다.

3. RFID – Internet of things sensing – Loyalty program

1. 지난 1년 매출액의 50%는 8월에 집중되어 있다.

: 시간에 따른 매출 추이를 정리한 내용으로, 데이터를 가공하여 흐름과 경향을 파악한 정보에 해당합니다. 데이터 간의 관계를 통해 의미를 도출하였으며, 정보를 구조화하거나 경험과의 결합은 따로 확인할 수 없으므로 정보임을 알 수 있습니다.

3. 날씨가 따뜻해지고, 지점을 확장하여 올 8월 매출액은 3000만원으로 예상한다.
: 다양한 정보를 바탕으로 미래의 매출을 예측하고 있으며, 이는 판단과 예측이라는 요소를 포함한 지혜에 해당됩니다. 정보나 지식을 넘어서, 복합적인 요인을 고려해 창의적 의사결정을 내리는 단계이기 때문에 다른 보기들과 구별됩니다.

4. 8월 A상품 구매 고객의 80%가 40대 여성 고객으로 대부분 회사원이다.

SQL에서 집계 함수는 여러 행을 대상으로 값을 요약하거나 계산하는 데 사용되며, 함수마다 적용 가능한 데이터의 자료형(type)이 다릅니다.

1. AVG
: AVG 함수는 평균을 계산하는 함수로, 숫자형(Numeric) 데이터에서만 사용이 가능합니다. 문자형이나 날짜형 데이터는 평균을 낼 수 없기 때문에 AVG 함수와 함께 사용 시 오류가 발생합니다.

2. COUNT
: COUNT 함수는 해당 컬럼에 값이 존재하는 행의 개수를 세는 함수로, 데이터의 타입에 관계없이 사용이 가능합니다. 문자형, 숫자형, 날짜형 등 모든 컬럼에 적용할 수 있으며, COUNT(*)는 NULL을 포함한 전체 행 수를, COUNT(column_name)은 해당 컬럼이 NULL이 아닌 행만을 계산합니다.

3. SUM
: SUM 함수는 숫자형 컬럼의 총합을 계산하는 함수로, 숫자형 데이터에서만 사용이 가능합니다. 문자형이나 날짜형 컬럼에는 합산 개념이 적용되지 않기 때문에 사용할 수 없습니다.

4. STDDEV
: STDDEV는 표준편차를 계산하는 함수로, 평균, 제곱, 분산 등의 연산이 포함되기 때문에 숫자형 데이터에만 사용이 가능합니다. 문자형이나 날짜형 데이터에는 수학적 연산이 불가능하므로 사용할 수 없습니다.

따라서, 문제의 정답은 '2. COUNT' 입니다.

고객관계관리(CRM, Customer Relationship Management)는 고객의 정보와 행동을 기반으로 관계를 강화하고, 장기적인 가치를 창출하기 위한 시스템입니다.

1. 부품의 설계, 제조, 유통 등의 공정 포함
: 이는 공급망 관리(SCM)와 관련된 설명입니다. CRM은 고객과의 접점을 관리하는 시스템으로, 제품 생산이나 유통 공정을 다루지 않습니다.

2. 외부 공급업체와의 정보시스템 통합으로 시간과 비용 최적화
: 이 설명은 공급망 관리(SCM)의 특징입니다. SCM은 공급자, 제조자, 유통업체 간의 통합을 통해 효율을 높이는 것을 목적으로 합니다. CRM은 내부 고객 정보 중심이므로 옳지 안은 설명입니다.

3. 기업의 내부 고객들만을 대상으로 한 정보시스템
: CRM은 외부 고객(소비자, 고객사 등)을 대상으로 관계를 유지하고 가치를 증대시키는 것이 목적입니다. 내부 고객을 대상으로 하는 것은 ERP(Enterprise Resource Planning)와 같은 내부 통합 시스템입니다.

4. 단순한 정보의 수집에서 탈피, 분석 중심의 시스템 구축 지향
: CRM은 단순히 고객 데이터를 수집하는 수준에서 벗어나, 고객 이탈 예측, 구매 패턴 분석, 세분화 마케팅 등 분석 기반의 전략적 시스템 구축을 목표로 합니다. 이는 CRM의 기능에 해당합니다.

비기봇 해설
인과관계(Causation)는 한 현상이 다른 현상의 원인이 되는 명확한 관계를 말하며, 원인과 결과가 분명히 정의됩니다. 상관관계(Correlation)는 두 현상 간에 통계적 연관이 있다는 것을 나타내지만, 이 중 어느 쪽이 원인인지, 혹은 단순히 우연적인 관계인지 알 수 없습니다.

1. (A): 상관관계, (B): 인과관계
: 상관관계는 단순히 두 현상 간의 연관성을 나타내는 것이므로, 원인과 결과를 포함하는 (A)에 부적합합니다. 예를 들어, 아이스크림 판매량과 익사 사고가 양의 상관관계를 가지지만, 둘 사이에 인과관계는 없습니다. 반대로, 인과관계는 (B)에 들어가면 "어느 쪽이 원인인지 알 수 없다"는 설명과 모순됩니다.

2. (A): 인과관계, (B): 상관관계
: 인과관계는 원인과 결과 간의 명확한 관계를 설명하므로 (A)에 적합합니다. 예를 들어, 운동이 심혈관 건강을 개선한다는 연구는 운동과 건강 사이에 인과관계를 나타냅니다. 반면, 상관관계는 두 현상이 통계적으로 연관되어 있지만, 원인과 결과를 파악할 수 없는 경우로, (B)에 적합합니다. 예를 들어, 키와 체중 사이에는 상관관계가 있지만, 키가 체중의 원인이라는 결론을 내릴 수 없습니다.

3. (A): 인과관계, (B): 선형관계
: 선형관계는 두 변수 간의 관계가 직선 형태로 표현되는 특정한 수학적 관계를 말합니다. 이는 상관관계의 한 형태이지만, 모든 상관관계가 선형적이지는 않습니다. (B)에 선형관계를 넣는 것은 상관관계를 일반화하지 못한 부적절한 설명입니다.

4. (A): 상관관계, (B): 선형관계
: 상관관계는 원인과 결과를 나타내지 않으므로 (A)에 적합하지 않습니다. 또한, 선형관계는 상관관계의 한 형태로, 상관관계 전체를 대표할 수 없으므로 (B)에도 부적합합니다.
