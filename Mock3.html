비기봇 해설
정답은 4번입니다.

SQL에서 그룹함수(집계함수)를 적용한 결과에 조건을 부여할 때는 HAVING 절을 사용합니다.

1. WHERE
: WHERE 절은 그룹화나 집계가 발생하기 이전에 개별 행(row)에 조건을 적용할 때 사용합니다. 예를 들어, 특정 범위의 데이터를 먼저 걸러낸 다음에 그룹화를 진행할 때 WHERE 절을 사용합니다. 그러나 그룹함수(예: SUM, AVG, COUNT 등)가 적용된 결과에는 WHERE 절을 사용할 수 없습니다.

2. ORDER BY
: ORDER BY절은 데이터의 정렬에 사용되며, 조건을 걸어 데이터를 필터링하는 기능은 수행하지 않습니다. 예를 들어, 결과를 오름차순(ASC) 또는 내림차순(DESC)으로 정렬할 때 사용됩니다.

3. GROUP BY
: GROUP BY 절은 데이터를 하나 이상의 컬럼을 기준으로 그룹화할 때 사용하며, 그룹화된 데이터에 대한 조건을 지정하는 기능은 제공하지 않습니다. 조건을 걸기 위해서는 HAVING 절을 사용해야 합니다.

4. HAVING
: HAVING 절은 GROUP BY 절에 의해 그룹화된 데이터 또는 그룹함수가 적용된 결과에 조건을 부여할 때 사용합니다. 예를 들어, 부서별 평균 급여가 3000 이상인 경우만 출력하고 싶을 때 HAVING 절을 사용합니다. WHERE 절로는 그룹화된 데이터의 조건을 지정할 수 없기 때문에 반드시 HAVING 절을 사용해야 합니다.

따라서, 문제의 정답은 '4. HAVING'입니다.

빅데이터의 출현 배경에는 데이터의 폭발적 증가(데이터 축적), 데이터 생성·저장 기술의 발전(휴대폰, 클라우드), 그리고 데이터를 효과적으로 처리·분석할 수 있는 기술 발전이 포함됩니다.

가. 대량의 데이터가 축적
: 산업 활동과 디지털 서비스의 확산으로 고객 데이터와 같은 대량의 데이터가 축적되었으며, 이러한 데이터는 기업과 기관이 새로운 가치를 발굴할 수 있는 자원이 되었습니다. 데이터 축적은 빅데이터 시대의 가장 기본적인 출현 배경입니다.

나. 휴대폰 및 클라우드의 발전
: 모바일 기기의 보급과 클라우드 컴퓨팅의 발전으로 데이터 생성과 저장이 용이해지고, 데이터 접근성이 비약적으로 향상되었습니다. 이러한 기술 발전은 방대한 데이터를 효과적으로 관리하고 활용할 수 있는 기반이 되었습니다.

다. 블록체인 기술의 탄생
: 블록체인은 데이터를 안전하게 기록하고 무결성을 보장하는 기술로, 빅데이터의 보안성과 투명성을 강화하기 위해 개발된 것입니다. 따라서 데이터 축적이나 처리 기술의 발전과는 성격이 다르며, 빅데이터의 출현 배경으로 보기 어렵습니다.

라. 분석처리 기술 발전
: 대규모 데이터를 처리하고 분석할 수 있는 아키텍처와 통계 도구들이 발전함으로써, 기업과 연구 기관은 방대한 데이터에서 의미 있는 정보를 효과적으로 추출할 수 있게 되었습니다. 분석 기술의 발전은 빅데이터 활용 가능성을 높인 핵심 요소입니다.

3. 운송 비용의 절감
: 물류 데이터를 분석하여 최적의 운송 경로와 방법을 도출하면 연료비와 인건비 등 운송 비용을 절감할 수 있습니다. 실제로 많은 유통·물류 기업들이 빅데이터를 활용하여 비용을 절감하고 있습니다.

4. 새로운 수익원의 발굴 및 활용

3. 백화점 고객을 세분화하여 다이렉트 마케팅 활용 – 유형분석
: 유형분석(클러스터링)은 데이터 내에서 유사한 특성을 가진 그룹을 식별하여 세분화하는 기법입니다. 백화점 고객을 세분화하여 특정 그룹에 맞는 마케팅 전략을 설계하는 데 사용됩니다. 대표적으로 K-평균 클러스터링 등이 활용됩니다.

4. 택배 경로를 최적화하기 위해 사용하는 방법 – 주성분분석
: 주성분분석(PCA)은 데이터의 차원을 축소하고 중요한 변수만 남겨 데이터의 구조를 단순화하는 데 사용됩니다. 이는 택배 경로 최적화와는 관련이 없으며, 택배 경로 최적화에는 주로 그래프 이론, 최단경로 알고리즘(예: 다익스트라 알고리즘), 또는 네트워크 최적화 기법이 사용됩니다.

: 개인정보 보호를 위해 데이터 접근을 제한하고, 필요 최소한의 정보만 접근하도록 통제하는 방식입니다. 개인정보 접근 제한은 개인정보 보호에 중요한 역할을 하지만, 데이터 오용 문제를 직접적으로 해결하는 방식은 아닙니다. 데이터가 이미 오용되고 있는 경우, 접근 제한만으로는 해결이 어렵습니다. 데이터 오용을 막기 위해서는 알고리즘의 처리 과정과 의사결정 구조를 검토할 수 있도록 하는 것이 보다 직접적인 해결책입니다.

4. 데이터 사용 기간 설정
: 데이터가 무기한 보관되는 것을 방지하고, 일정 기간이 지나면 폐기하도록 설정하는 방식입니다. 데이터 사용 기간을 설정하는 것은 보조적인 해결책일 수 있지만, 데이터 오용을 직접적으로 방지하는 방법은 아닙니다. 사용 기간을 설정하더라도 그 기간 내에서 데이터가 잘못 사용될 가능성이 존재합니다.

비기봇 해설
1. 문제 조건 정리
- 확률변수 X의 가능한 값: 1,2,4
- 확률: p(X=1)=0.4
- 기댓값: E(X)=2.4

확률의 합은 반드시 1이므로 다음과 같이 식을 세울 수 있습니다.
p(X=1)+p(X=2)+p(X=4)=1

이를 대입하여 정리하면 0.4+p(X=2)+p(X=4)=1이 되고, 따라서 p(X=2)+p(X=4)=0.6 (식 (1)) 이 됩니다.

2. 기댓값 공식 적용
기댓값 E(X)는 확률과 값의 곱의 합으로 정의됩니다. 따라서 E(X)는 아래와 같이 계산됩니다.
E(X)=∑[x⋅p(x)]=1⋅p(X=1)+2⋅p(X=2)+4⋅p(X=4)

이를 주어진 값에 대입하면 2.4=1⋅0.4+2⋅p(X=2)+4⋅p(X=4)가 됩니다. 이를 정리하면 다음과 같습니다.
2.4=0.4+2⋅p(X=2)+4⋅p(X=4)⇒2⋅p(X=2)+4⋅p(X=4)=2 (식 (2))

3. 연립 방정식 풀이
식 (1)에서 p(X=4)를 0.6−p(X=2)로 나타낼 수 있으며, 이를 식 (2)에 대입하면 다음과 같습니다.
2⋅p(X=2)+4⋅(0.6−p(X=2))=2

괄호를 풀고 정리하면
2⋅p(X=2)+2.4−4⋅p(X=2)=2
−2⋅p(X=2)=−0.4⇒p(X=2)=0.2 가 됩니다.

따라서 p(X=2)=0.2 임을 알 수 있습니다.

비기봇 해설
회귀분석에서 후진제거법은 모든 독립변수를 포함한 모형에서 시작하여 가장 적은 영향을 주는 변수를 하나씩 제거하는 방법입니다. 변수를 제거하면서 모형의 유의성을 유지하는 것이 목표이며, 더 이상 유의하지 않은 변수가 없을 때 모형을 완성합니다. 이 방법은 데이터에 포함된 많은 변수를 시작점으로 활용하므로, 초기 모형이 복잡할 때 유용합니다.

1. 후진제거법 (Backward Elimination)
: 후진제거법은 모든 변수를 포함한 상태에서 출발하여, 유의하지 않은 변수를 제거하는 방식입니다. 변수가 많을 때 초기 모형에서 출발하여 단계적으로 모형을 단순화하므로, 주어진 설명과 가장 적합한 방법입니다.

2. 전진선택법 (Forward Selection)
: 전진선택법은 후진제거법과 반대 방식으로, 처음에는 변수를 포함하지 않은 상태에서 시작합니다. 가장 유의미한 변수부터 하나씩 추가하면서 모형을 확장해 나가는 방식입니다. 문제에서 제시한 설명과는 다릅니다.

3. 단계적 회귀법 (Stepwise Regression)
: 단계적 회귀법은 전진선택법과 후진제거법을 결합한 방식입니다. 변수를 추가하거나 제거하는 과정을 반복하여, 모형의 최적화를 동시에 추구합니다. 문제에서 제시된 설명은 순수한 후진제거법을 설명하므로, 이 방법은 적합하지 않습니다.

4. 교차검증 (Cross Validation)
: 교차검증은 회귀모형의 성능을 평가하거나 과적합을 방지하기 위해 데이터를 여러 부분으로 나누어 검증하는 방법입니다. 이는 변수 선택이 아니라 모형의 일반화 성능을 확인하는 과정에 사용되므로 문제의 설명과는 관련이 없습니다.

- FP (False Positive): 300 (False를 True로 잘못 예측한 값)

=100400=0.25
