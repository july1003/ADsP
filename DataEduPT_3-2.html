>"+"(2, 3)
[1] 5

2. 새로운 워크스페이스에서 다음의 명령어들을 수행시켰다. 이때 에러를 생성하는 명령어는?
  1. c(2, 1, TRUE)
  2. c(2, ab, cd)
  3. c(pi, 2*pi)
  4. c(FALSE, 3, 5)
정답보기
정답
2 c(2, ab, cd)

해설
c(2, ab, cd)는 에러를 발생시킨다. R에서 벡터는 동일한 타입의 원소로 구성되어야 하며, ab와 cd는 문자열이기 때문에 인용부호(" ")가 필요하다. 인용부호 없이 작성하면 에러가 발생한다. 나머지 명령어 c(2, 1, TRUE)와 c(FALSE, 3, 5)는 논리형이 자동으로 숫자형으로 변환되고, c(pi, 2*pi)는 pi가 숫자형 상수이므로 정상적으로 수행된다.
비기봇에게 문의중…
비기봇 해설
명령어 실행 결과는 아래와 같습니다.

1. c(2, 1, TRUE)
• 결과
[1] 2 1 1
: 벡터 c() 함수는 가장 큰 데이터 타입으로 자동 변환(Coercion)합니다. 2 (numeric), 1 (numeric), TRUE (logical)이므로, TRUE가 1로 변환됩니다.

2. c(2, ab, cd)
: ab, cd는 정의되지 않은 변수이므로, 실행 시 "object 'ab' not found" 에러가 발생합니다. 만약 "ab", "cd"처럼 문자열로 감싸면 정상적으로 실행됩니다.

3. c(pi, 2*pi)
• 결과
[1] 3.141593 6.283185
: pi는 R에서 미리 정의된 상수(3.141592...)이며, 2*pi도 유효한 연산입니다.

4. c(FALSE, 3, 5)
• 결과
[1] 0 3 5
: FALSE는 0으로 변환되며, 숫자 벡터가 생성됩니다.

따라서 정답은 2. c(2, ab, cd)입니다.

3. 아래 그림과 같이 두개의 데이터 프레임 dfm1, dfm2 를 T_name 이라는 변수로 결합하고자 할 때, 사용되는 함수는 어느 것인가?
  1. cbind(dfm1, dfm2, by=“T_name”)
  2. rbind(dfm1, dfm2, by=“T_name”)
  3. merge(dfm1, dfm2, by=“T_name”)
  4. subset(dfm1, dfm2, by=“T_name”)
정답보기
정답
3 merge(dfm1, dfm2, by=“T_name”)

해설
두 개의 데이터 프레임을 특정 키(T_name)를 기준으로 결합하는 함수는 merge() 함수이다. merge() 함수는 R에서 두 개의 데이터 프레임을 병합할 때 사용되며, 공통된 키 변수를 기준으로 데이터를 결합한다. cbind() 함수는 데이터 프레임의 열을 기준으로 병합할 때 사용한다. rbind() 함수는 데이터 프레임의 행을 추가할 때 사용한다. subset() 함수는 데이터 프레임의 일부를 선택할 때 사용한다.

비기봇 해설
데이터 프레임을 특정 열을 기준으로 결합할 때 사용하는 함수는 merge입니다. 이는 두 데이터 프레임을 공통 열을 기준으로 병합하여 하나의 데이터 프레임으로 만드는 데 사용됩니다.

6. 여섯 가지 닭 사료 첨가물의 효과를 비교한 데이터에 대한 요약 통계 결과이다. 다음 중 데이터 요약에 대한 설명으로 부적절한 것은?
  1. Weight의 중앙값은 261.3이다.
  2. feed는 닭 사료 첨가물 종류를 나타내며 범주형 변수이다.
  3. 약 25%의 닭의 weight가 204.5보다 작다.
  4. weight의 범위는 최대값과 최소값의 차이로, 315이다.
정답보기
정답
1 Weight의 중앙값은 261.3이다.

해설
Weight의 중앙값은 summary 결과에서 258.0으로 나타나 있다. 중앙값은 데이터를 정렬했을 때 중간에 위치한 값으로, Median으로 표현하며, 평균(mean)과는 다른 개념이다. 따라서 "Weight의 중앙값은 261.3이다"라는 설명은 부적절하다. feed 변수는 닭 사료 첨가물의 종류를 나타내며 범주형 변수라는 설명은 옳은 설명이며, 약 25%의 닭의 weight가 204.5보다 작다는 설명도 1사분위수(1st Qu.)를 기준으로 하므로 정확하다. 또한 weight의 범위를 최대값과 최소값의 차이로 계산하면, 423.0 - 108.0 = 315.0이다.
비기봇에게 문의중…
비기봇 해설
summary 함수는 데이터의 요약 통계량을 제공합니다. 연속형 변수는 최소값, 최대값, 중앙값(Median), 평균(Mean), 1사분위수(1st Quartile), 3사분위수(3rd Quartile)를 보여주며, 범주형 변수는 각 범주별 데이터 빈도를 나타냅니다.

1. Weight의 중앙값은 261.3이다.
: summary 결과에서 weight의 중앙값(Median)은 258.0으로 표시되어 있습니다. 중앙값은 데이터를 오름차순으로 정렬했을 때 중간에 위치한 값을 나타냅니다. 따라서 중앙값이 261.3이라는 설명은 잘못된 정보입니다.

2. feed는 닭 사료 첨가물 종류를 나타내며 범주형 변수이다.
: summary 결과에서 feed는 닭 사료 첨가물 종류를 나타내며, casein, horsebean, linseed 등 총 6개의 범주로 구성되어 있습니다. 각 범주의 빈도(casein: 12, horsebean: 10 등)가 표시되어 있으므로 feed가 범주형 변수임을 명확히 알 수 있습니다.

3. 약 25%의 닭의 weight가 204.5보다 작다.
: summary 결과에서 weight의 1st Qu. 값이 204.5로 나타나 있습니다. 1사분위수(1st Quartile)는 데이터의 하위 25%가 해당 값보다 작거나 같음을 의미합니다. 따라서 약 25%의 닭의 weight가 204.5보다 작다는 설명은 정확합니다.

4. weight의 범위는 최대값과 최소값의 차이로, 315이다.
: summary 결과에서 weight의 최대값은 423.0, 최소값은 108.0으로 나타나 있습니다. 범위(Range)는 데이터의 최대값에서 최소값을 뺀 값으로, 계산해보면 423.0 - 108.0 = 315.0이 됩니다. 따라서 weight의 범위는 315입니다.

7. Carseats 데이터 프레임은 400개 상점에서 판매 중인 유아용 카시트의 재료이고, Sales 변수는 해당 상점에서 판매된 카시트의 수를 나타낸다. 다음 중 R 패키지에서 Sales 변수의 표준편차를 계산하기 위한 식으로 가장 부적절한 것은?
  1. stdev(Carseats$Sales)
  2. sd(Carseats$Sales)
  3. sqrt(var(Carseats$Sales))
  4. var(Carseats$Sales)^(1/2)
정답보기
정답
1 stdev(Carseats$Sales)

해설
R에서 표준편차를 계산하는 함수는 sd()가 표준이며, sqrt(var()) 역시 분산의 제곱근을 이용해 표준편차를 계산할 수 있다. 그러나 stdev()는 R의 기본 패키지에 존재하지 않는 함수이므로 에러가 발생한다.
비기봇에게 문의중…
비기봇 해설
데이터 분석에서 표준편차를 계산하기 위해 R에서 주로 사용하는 함수는 sd()입니다.
1. stdev(Carseats$Sales): R에서는 stdev()라는 함수를 제공하지 않으므로, 이 식은 오류를 발생시킵니다.
2. sd(Carseats$Sales): R에서 표준편차를 계산하는 데 사용되는 올바른 함수입니다.
3. sqrt(var(Carseats$Sales)): 분산을 계산한 후 제곱근을 취하여 표준편차를 구하는 방법으로, 올바른 접근입니다.
4. var(Carseats$Sales)^(1/2): 분산의 제곱근을 취하여 표준편차를 구하는 방법으로, 올바른 접근입니다.
