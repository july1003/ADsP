비기봇 해설
1. 페타바이트 (Petabyte, PB)
: 1,0245 Byte로, 약 1,000 테라바이트(TB)에 해당하는 데이터 크기 단위입니다. 페타바이트는 대규모 데이터 저장소에서 사용되며, 기업의 데이터 아카이브, 대형 연구 프로젝트, 그리고 영상 데이터 저장 등에 널리 활용됩니다. 예를 들어, 영화 스트리밍 서비스에서 수천 개의 고화질(HD) 영화 파일을 저장하는 데 필요한 용량을 나타낼 때 자주 사용됩니다.

2. 엑사바이트 (Exabyte, EB)
: 1,0246 Byte로, 대규모 데이터 센터의 총 데이터 용량이나 기업 전체의 데이터 저장 규모를 나타낼 때 사용됩니다. 또한, 클라우드 저장 용량, 빅데이터 분석, 그리고 글로벌 인터넷 트래픽 측정을 위한 단위로 자주 사용됩니다.

3. 제타바이트 (Zettabyte, ZB)
: 1,0247 Byte로, 전 세계의 인터넷 트래픽, 클라우드 데이터 저장 용량, 데이터 분석 및 IoT 시대의 초대규모 데이터를 표현하는 데 사용됩니다. 2020년 기준 전 세계 클라우드 저장 용량이 약 40제타바이트(ZB)에 도달한 것으로 추정되며, 이는 전 세계적으로 엄청난 데이터의 증가를 보여줍니다. 제타바이트는 엑사바이트(EB)의 1,024배로, 대규모 데이터 환경에서 점점 더 일반화되고 있는 단위입니다.

4. 요타바이트 (Yottabyte, YB)
: 1,0248 Byte로, 현재 이론적으로 가장 큰 데이터 단위입니다. 현재까지 실질적으로 사용되지 않는 단위로, 초대규모 데이터를 저장하거나 초거대 인터넷 트래픽을 상정할 때 이론적으로만 사용됩니다. 미래 활용 가능성으로는 전 세계적으로 생성되는 모든 데이터의 총합을 표현하거나, 사물인터넷(IoT)과 초연결 네트워크 시대에서 발생하는 데이터를 저장하는 데 쓰일 수 있습니다.

따라서, 데이터 단위를 작은 것에서 큰 것 순으로 나열해보면 정답은 '2. 페타바이트 < 엑사바이트 < 제타바이트 < 요타바이트'가 됩니다.

정답은 3번입니다.

• 정성적 데이터: 형태와 형식이 명확하지 않고 언어나 문자 등으로 표현됩니다.

1. 풍향
: 풍향은 바람이 불어오는 방향을 의미하며, 동쪽, 서쪽과 같은 방향으로 표현될 수 있으나 일반적으로 0~360도의 각도로 수치화되어 사용됩니다. 따라서 정량 데이터로 구분됩니다.

2. 습도
: 습도는 공기 중 수증기의 상대적인 양을 백분율(%)로 나타내며, 명확한 수치로 표현 가능하므로 정량 데이터에 속합니다.

3. 기상특보
: 기상특보는 "호우주의보", "폭염경보" 등과 같이 문자 또는 언어로 표현되는 질적인 정보이다. 수치화되지 않으며, 상황에 따라 해석이 달라질 수 있는 정성 데이터에 해당합니다.

4. 1시간 강수량
: 일정 시간 동안 내린 비의 양을 밀리미터(mm) 단위로 측정하는 수치 데이터로, 정량 데이터에 해당됩니다.

따라서, 문제의 정답은 '3. 기상특보' 입니다.

비기봇 해설
암묵지와 형식지 간의 상호작용은 4단계로 나누어 설명할 수 있습니다.
1. 공통화: 암묵지 → 암묵지
2. 표출화: 암묵지 → 형식지
3. 연결화: 형식지 → 형식지
4. 내면화: 형식지 → 암묵지

(가) 공통화(Socialization) - (A) 개인 간의 대화를 통해 경험과 감각적 지식을 직접적으로 공유함
: 공통화는 개인 간의 상호작용을 통해 암묵적 지식을 공유하는 과정입니다. 이 과정은 문서화나 구조화되지 않은 경험과 감각적 지식을 기반으로 이루어지며, 직접적인 대화나 관찰, 실습 등을 통해 지식이 전달됩니다.

(나) 표출화(Externalization) - (C) 축적된 암묵적 경험을 체계화하여 매뉴얼이나 보고서로 전환함
: 표출화는 개인의 암묵적 경험을 문서화하여 형식적 지식으로 전환하는 과정입니다. 조직 내에서 보고서, 매뉴얼, 절차 등을 통해 경험을 구조화함으로써 다른 사람들이 활용할 수 있도록 지식을 공유합니다.

(다) 내면화(Internalization) - (B) 문서화된 지식을 학습하여 자신의 직무나 작업에 적용할 수 있는 방식으로 이해함
: 내면화는 문서화된 형식적 지식을 학습하고 이를 개인의 암묵적 지식으로 내재화하는 과정입니다. 개인은 매뉴얼이나 보고서를 통해 학습한 내용을 실제 업무에 적용하며, 새로운 경험을 통해 암묵적 지식이 축적됩니다.

(라) 연결화(Combination) - (D) 여러 개의 문서를 결합하여 새로운 지식 체계를 생성함
: 연결화는 이미 존재하는 형식적 지식을 조합하여 새로운 지식 체계를 구축하는 과정입니다. 다양한 데이터, 문서, 보고서를 통합하고 이를 바탕으로 새로운 통찰이나 지식을 창출합니다.

따라서 정답은 '④ (가) - (A), (나) - (C), (다) - (B), (라) - (D)'입니다.

비기봇 해설
DIKW 피라미드는 데이터, 정보, 지식, 지혜의 순서로 계층화된 모델입니다. "근본 원리에 대한 깊은 이해를 바탕으로 도출되는 창의적 아이디어"는 지식과 아이디어가 결합된 창의적인 산물로, DIKW 피라미드의 최상위 계층인 "지혜"에 해당합니다.

1. 데이터 (Data)
: 데이터는 관찰하거나 수집된 단순한 사실이나 수치로, 해석이나 분석이 이루어지지 않은 원시 형태의 값을 말합니다. 보기에서 제시된 "근본 원리에 대한 깊은 이해"와 창의적인 산물은 단순한 데이터의 범위를 넘어서므로 데이터는 적합하지 않습니다.

2. 정보 (Information)
: 정보는 데이터를 분석하거나 정리하여 의미를 부여한 결과입니다. 정보는 여전히 단순한 해석의 단계에 머물며, 보기에서 언급된 창의적 아이디어나 깊은 이해는 정보 단계에서 다룰 수 없는 부분입니다.

3. 지식 (Knowledge)
: 지식은 정보를 바탕으로 문제를 해결하거나 의사결정을 내릴 수 있는 상태를 의미하며, 근본 원리를 이해하고 이를 축적한 상태라면 지식으로 볼 수 있습니다. 그러나 "창의적 아이디어"와 "지식의 축적과 결합된 창의적 산물"은 지식 단계를 넘어 지혜의 단계에 해당하므로 지식에 적합하지 않습니다.

4. 지혜 (Wisdom)
: 지혜는 지식을 통합하고 근본 원리에 대한 깊은 이해를 바탕으로 새로운 통찰이나 창의적인 아이디어를 도출하는 단계입니다. 보기에서 언급된 "지식의 축적과 아이디어가 결합된 창의적인 산물"과 "근본 원리에 대한 깊은 이해"는 지혜를 가장 잘 설명하는 요소들입니다.

비기봇 해설
정답은 2번입니다.

DIKW 피라미드는 데이터 분석 및 정보 활용의 수준을 구분하는 체계로, 데이터(Data)→정보(Information)→지식(Knowledge)→지혜(Wisdom)의 구조를 가집니다.

1. A사이트 보다 B사이트가 다른 물건도 비싸게 팔 것이다.
: 이는 지혜(Wisdom)에 해당되는 설명입니다. A사이트와 B사이트의 가격 정보를 바탕으로 추정을 하고 있습니다. 하지만 구체적인 행동(무엇을 구매하겠다 등)은 포함되지 않았으므로 지식에 해당하지 않습니다.

2. B사이트 보다 가격이 상대적으로 저렴한 A사이트에서 USB를 사야겠다.
: 단순 가격 정보(Information)를 바탕으로 구체적인 행동(USB 구매 결정) 으로 이어지고 있습니다. 정보가 해석되고, 의사결정이 포함되었으므로 지식(Knowledge) 에 해당됩니다.

비기봇 해설
정답은 3번입니다.

SQL은 사용 목적에 따라 DML(데이터 조작어), DDL(데이터 정의어), DCL(데이터 제어어), TCL(트랜잭션 제어어)로 구분됩니다.

A. UPDATE
: DML에 해당합니다. 테이블에 저장된 데이터를 수정하는 명령어입니다. 데이터 값을 변경하므로 데이터 조작어에 포함됩니다.

B. SELECT
: DML에 해당합니다. 테이블에서 데이터를 조회하는 명령어입니다. 비록 데이터의 내용은 변경되지 않지만, 데이터베이스에서 데이터를 가져와서 사용하기 때문에 데이터 조작어(DML)에 포함됩니다.

C. INSERT
: DML에 해당합니다. 테이블에 새로운 데이터를 추가(삽입)하는 명령어입니다.

D. DELETE
: DML에 해당합니다. 테이블에서 데이터를 삭제하는 명령어입니다.

E. CREATE
: DDL에 해당합니다. 테이블, 뷰, 인덱스 등 데이터베이스 구조(스키마)를 생성하는 명령어입니다. 데이터 자체가 아닌 데이터 구조를 정의하므로 DML이 아닙니다.

따라서, 문제의 정답은 '3. A, B, C, D' 입니다.

데이터마스킹은 특정 데이터 값을 삭제하는 것이 아니라, 대체하거나 가려서 식별 가능성을 줄이는 방법입니다.

1. 가명처리 – 데이터의 특정 식별 정보를 직접적으로 알 수 없는 값으로 변경
: 가명처리는 개인정보 비식별화의 대표적인 기법 중 하나로, 데이터를 식별할 수 없도록 특정 정보를 임의의 값으로 대체하는 방식입니다. 다만, 가명처리는 원본 데이터와의 연계 가능성을 완전히 제거하지 않기 때문에 재식별 가능성이 존재할 수 있습니다.

2. 범주화 – 데이터의 식별 정보를 해당 데이터가 속하는 범주나 그룹으로 변환
: 범주화는 데이터의 세부적인 값을 그룹화하거나 범주로 변환하여 특정 개인을 식별할 수 없게 하는 기법입니다. 예를 들어, 특정 나이를 10대, 20대와 같이 범주로 나타내는 방식입니다.

3. 데이터마스킹 – 개인 정보 식별이 가능한 특정 데이터 값 삭제 처리
: 데이터 마스킹은 데이터를 삭제하는 것이 아니라, 특정 데이터를 가공하여 식별 가능성을 낮추는 방법을 말합니다. 예를 들어, 이름의 일부를 별표(*)로 대체하거나 임의의 값으로 변경하여 사용합니다. 삭제와는 본질적으로 다르며, ‘삭제 처리’라는 설명은 데이터 마스킹에 대한 잘못된 설명입니다.

4. 총계처리 - 데이터 값을 집계 값으로 대체하여 원본 데이터를 숨기는 방법
: 총계처리는 데이터를 요약하거나 집계 값을 사용하여 원본 데이터를 보호하는 기법입니다. 이는 개별 데이터를 숨기면서도 분석 목적에 맞게 활용 가능하게 합니다.

비기봇 해설
소프트 스킬(Soft Skills)은 기술적 능력(하드 스킬)과 달리, 업무의 효율성과 협업을 높이는 데 필요한 개인적·대인적 역량입니다. 데이터 사이언티스트는 데이터 분석 기술뿐만 아니라, 결과를 해석하고 의사소통하며 문제를 창의적으로 해결할 수 있는 소프트 스킬이 필수적입니다.

1. 이론적 지식
: 이론적 지식은 데이터 사이언티스트가 데이터를 분석하고 모델을 개발하기 위해 필요한 기술적 역량(하드 스킬)에 해당합니다. 이는 통계학, 머신러닝, 데이터 엔지니어링 등 전문적인 기술적 배경을 의미하며, 소프트 스킬로 분류하기는 어렵습니다.

2. 시각화 활용 능력
: 시각화 활용 능력은 데이터를 이해하기 쉽도록 전달하는 데 중요한 소프트 스킬입니다. 데이터 사이언티스트는 데이터의 인사이트를 명확히 전달하기 위해 적절한 시각화 도구와 기법을 활용할 수 있어야 합니다. 소프트 스킬로 적절한 설명입니다.

3. 커뮤니케이션 능력
: 커뮤니케이션 능력은 데이터 분석 결과를 비기술적 이해관계자에게 전달하거나, 팀원들과 협력하여 문제를 해결하는 데 중요한 소프트 스킬입니다. 데이터 사이언티스트가 반드시 갖추어야 할 역량 중 하나입니다.

4. 창의적 사고
: 창의적 사고는 비즈니스 문제를 해결하기 위해 새로운 접근법을 시도하고, 데이터로부터 독창적인 인사이트를 도출하는 데 중요한 소프트 스킬입니다. 데이터 사이언티스트의 역할에 필수적인 역량으로 적절한 설명입니다.

비기봇 해설
전략적 통찰력은 데이터를 기반으로 조직의 장기적인 목표와 비전을 이해하고, 이를 효과적으로 달성하기 위한 방향을 제시하는 데 중점을 둡니다. 이는 단기적인 문제 해결이 아닌, 조직의 핵심 가치와 비전을 반영하여 분석 지표를 설계하고 의사결정을 객관적으로 지원하는 과정을 포함합니다.

1. 조직의 비전을 이해하고 분석에 이를 반영한다.
: 전략적 분석의 핵심은 조직의 장기적인 비전과 목표를 이해하고, 이를 데이터 분석 과정에 반영하는 것입니다. 이를 통해 분석 결과가 조직의 방향성과 일치하며, 전략적 의사결정을 지원할 수 있습니다.

2. 데이터 기반 의사결정을 통해 객관적 판단을 내린다.
: 데이터 기반 의사결정은 전략적 통찰력을 얻기 위한 기본적인 접근 방식입니다. 주관적인 판단이 아닌, 객관적이고 신뢰할 수 있는 데이터에 기반하여 결정을 내리는 것은 전략적 분석의 필수 요소입니다.

3. 사업 상황을 확인하기 위해 사업 내부의 문제들을 집중하여 분석을 이용한다.
: 사업 내부 문제에 초점을 맞추는 것은 단기적 문제 해결을 위한 전술적 분석에 해당합니다. 전략적 분석은 내부 문제에 집중하기보다는 전체적인 사업 환경과 비전을 고려하며, 장기적인 성공을 위한 통찰력을 제공합니다.

4. 핵심가치와 연관된 분석 지표를 설계하고 지속적으로 관리한다.
: 핵심가치와 연관된 지표를 설계하고 관리하는 것은 전략적 분석의 중요한 부분입니다. 이를 통해 조직의 성과를 지속적으로 모니터링하고, 핵심 가치와 목표 달성 여부를 평가할 수 있습니다.

비기봇 해설
• 디지털화(Digitalization)
: 과거에는 아날로그 데이터를 디지털로 변환하여 데이터를 저장, 처리, 관리하는 데 중점을 두었습니다. 이는 기술적 효율성을 높이고 정보의 활용 가능성을 극대화하는 단계입니다. 디지털화는 데이터의 구조적 변환을 통해 정보 관리의 기초를 형성하며, 빅데이터 시대의 출발점입니다.

• 연결(Connection)
: 현재는 디지털화된 데이터를 서로 연결하는 단계로, 네트워크와 플랫폼 기반 경제가 주요 기반이 됩니다. 디지털화된 정보와 시스템이 연결됨으로써 데이터의 가치는 더욱 확대됩니다. 연결 단계에서는 정보의 상호작용과 통합이 이루어지며, 이를 통해 더 큰 가치를 창출할 수 있는 기반을 마련합니다.

• 에이전시(Agency)
: 미래에는 연결된 데이터를 활용하여 자동화된 의사결정과 개인화된 서비스를 제공하는 에이전시 단계로 발전합니다. 이는 데이터를 기반으로 자율적이고 신뢰할 수 있는 시스템을 통해 개인화된 경험과 서비스를 제공하는 것을 목표로 합니다. 에이전시 단계는 데이터의 최종 활용 단계로, 복잡한 데이터 구조를 기반으로 효율적이고 스마트한 의사결정을 가능하게 합니다.
